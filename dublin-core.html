Микроразметка Dublin Core

Dublin Core — микроразметка, которая обеспечивает тематическое разделение страниц сайта и веб-приложений. Как настроить микроразметку Dublin Core читайте внутри статьи.


Микроразметка Dublin Core была создана в 1995 г. для формирования большой библиотеки сайтов во всемирной паутине. Разметка создана для стандартизации всемирной паутины. Позволяет тематически разделить всей сайты и их страницы во всемирной паутине.


Dublin Core разработана международной междисциплинарной группой профессионалов библиотечного дела, компьютерных наук, кодирования текстов, музейного дела и других смежных групп.


Основные теги Dublin Core

  • DC.title – Заголовок. Аналогичен стандартному мета-тегу Title
  • oDC.creator – Автор проекта. Лицо, несущее ответственность за создание веб-сайта
  • DC.subject – Ключевые слова страницы. Аналогичны мета-тегу keywords. Допускается наличие нескольких тегов subject, которые содержат разные ключевые слова
  • DC.description – Описание страницы. Аналогичен стандартному мета-тегу description. Содержимое тега может не совпадать с мета-тегом description. Описание может быть любого размера
  • DC.publisher – Издатель. Лицо, несущее ответственность за предоставление ресурса пользователям. Может принимать значения: ФИО человека, названия организации или службы
  • DC.publisher.url – Полная ссылка на сайт издателя

Все элементы набора факультативны и повторяемы. Элементы метаданных могут появляться в любом порядке.
